# BP设备 BLE 协议与嵌入式实现说明（esp-idf esp32s3）

## 1. 服务与特征定义

- **主服务 UUID**：`6E400001-B5A3-F393-E0A9-E50E24DCCA9E`
- **命令特征（Write/WriteWithoutResponse）UUID**：`6E400003-B5A3-F393-E0A9-E50E24DCCA9E`
- **状态/数据特征（Notify）UUID**：`6E400004-B5A3-F393-E0A9-E50E24DCCA9E`
- **设备信息特征（Read）UUID**：`6E400005-B5A3-F393-E0A9-E50E24DCCA9E`

## 2. 数据包结构

所有通信数据包均采用如下结构（字节序为小端）：

| 字段         | 长度 | 说明                   |
| ------------ | ---- | ---------------------- |
| 协议版本     | 1    | 固定 0x01              |
| 命令类型     | 1    | 见下表                 |
| 序列号       | 2    | 递增或时间戳           |
| 颜色数量     | 1    | n                      |
| RGB颜色序列  | 3*n  | 每色3字节              |
| 震动模式     | 1    | 0x00~0x06              |
| 震动强度     | 1    | 0x01~0x03              |
| 文本长度     | 1    | 最多64字节             |
| 文本内容     | n    | UTF-8编码              |
| 持续时间     | 2    | 毫秒                   |
| 校验和       | 1    | 前面所有字节累加和&0xFF |

## 3. 命令类型定义

| 类型名           | 值   | 说明           |
| ---------------- | ---- | -------------- |
| notification     | 0x01 | 通知消息       |
| rgbControl       | 0x02 | RGB灯光控制    |
| vibrationControl | 0x03 | 震动控制       |
| textDisplay      | 0x04 | 文本显示       |
| deviceStatus     | 0x05 | 设备状态查询   |
| batteryLevel     | 0x06 | 电池电量查询   |

## 4. 典型指令与推送

### 4.1 控制端下发（Write）

- **发送通知**：组包后写入命令特征
- **查询状态/电量**：发送对应命令包，设备收到后应通过 Notify 回复

### 4.2 设备端推送（Notify）

- **测量结果/状态/告警**：组包后通过 Notify 主动推送到状态特征

## 5. 典型流程

1. **连接建立**：App 连接设备，发现服务与特征
2. **使能 Notify**：App 订阅状态特征
3. **业务交互**：App 发送命令，设备解析并执行，必要时通过 Notify 推送结果
4. **断开连接**：App 或设备主动断开

## 6. 实现建议（esp-idf）

- 使用 `NimBLE` 或 `bluedroid` BLE GATT 服务实现
- 按上述 UUID 注册服务与特征，命令特征支持 Write/WriteWithoutResponse，状态特征支持 Notify
- 收到命令特征写入时，按协议解析数据包并执行相应动作
- 业务事件（如测量完成、异常、低电量）主动组包通过 Notify 推送
- 校验和错误、非法包应丢弃并可选上报异常
- 推荐单连接模式，连接新主机时可主动断开旧连接

## 7. 注意事项

- 文本最大64字节，超长需截断
- 颜色数量建议≤20，超出可忽略多余
- 所有多字节字段均小端序
- 校验和为前面所有字节累加和的低8位
- 建议支持OTA升级与设备信息读取
