## 架构概览

Bipupu 采用前后端分离的架构设计：

- **后端**: FastAPI 提供 RESTful API 和 WebSocket 服务
- **移动端**: Flutter 跨平台应用
- **数据库**: PostgreSQL 主数据库 + Redis 缓存
- **消息队列**: Celery 处理异步任务
- **部署**: Docker 容器化部署

### 架构特点

- ✅ **高内聚低耦合**: 模块化设计，职责清晰
- ✅ **水平扩展**: 支持多实例部署
- ✅ **实时通信**: WebSocket 实现消息实时推送
- ✅ **离线优先**: 移动端支持离线语音识别和合成
- ✅ **安全**: JWT 认证 + Argon2 密码加密

### 核心表结构

#### users 表

```sql
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    username VARCHAR(50) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    nickname VARCHAR(50),
    bipupu_id VARCHAR(100) UNIQUE NOT NULL,
    avatar_data BYTEA,
    timezone VARCHAR(50) DEFAULT 'UTC',
    birth_date DATE,
    zodiac_sign VARCHAR(20),
    chinese_zodiac VARCHAR(10),
    mbti VARCHAR(4),
    is_active BOOLEAN DEFAULT TRUE,
    is_superuser BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### messages 表

```sql
CREATE TABLE messages (
    id SERIAL PRIMARY KEY,
    sender_bipupu_id VARCHAR(100) NOT NULL,
    receiver_bipupu_id VARCHAR(100) NOT NULL,
    content TEXT NOT NULL,
    message_type VARCHAR(20) DEFAULT 'normal',
    pattern JSONB,
    waveform INTEGER[],
    is_read BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_receiver_created ON messages(receiver_bipupu_id, created_at);
CREATE INDEX idx_sender_created ON messages(sender_bipupu_id, created_at);
```

### 安全措施

| 层级 | 措施 | 实现 |
|------|------|------|
| **传输层** | HTTPS/TLS | Nginx 反向代理 |
| **认证层** | JWT + Bearer | python-jose |
| **密码层** | Argon2 哈希 | passlib |
| **API 层** | 频率限制 | slowapi |
| **数据层** | 参数化查询 | SQLAlchemy ORM |
| **文件层** | 类型/大小验证 | 自定义验证器 |

## 性能优化

### 缓存策略

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   Client    │────▶│    API      │────▶│    Redis    │
│   Request   │     │   Server    │     │   Cache     │
└─────────────┘     └──────┬──────┘     └──────┬──────┘
                           │                    │
                           │ Cache Miss         │
                           ▼                    │
                    ┌─────────────┐             │
                    │ PostgreSQL  │─────────────┘
                    │   (DB)      │    Cache Fill
                    └─────────────┘
```

### 优化措施

| 方面 | 措施 | 效果 |
|------|------|------|
| **数据库** | 连接池 (pool_size=50) | 支持高并发 |
| **数据库** | 复合索引 | 查询加速 |
| **缓存** | Redis + 内存缓存 | 减少 DB 查询 |
| **API** | 增量同步 (since_id) | 减少数据传输 |
| **API** | 长轮询 (Long Polling) | 实时性提升 |
| **静态文件** | ETag 缓存 | 减少带宽 |
